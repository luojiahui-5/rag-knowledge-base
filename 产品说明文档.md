# 企业级智能知识库 RAG 系统 — 产品说明文档

> 版本：v1.0.0 | 更新日期：2026-07-20

---

## 一、产品概述

企业级智能知识库系统是一套基于 **RAG（Retrieval-Augmented Generation）** 技术的智能知识管理平台。系统将企业内部文档（PDF、Word、PPT、Markdown 等）自动解析、切片、向量化，结合 DeepSeek 大语言模型，实现**自然语言问答**——员工用日常语言提问，系统直接从企业文档中找到答案并生成回复，附来源引用。

### 核心价值

| 场景 | 传统方式 | 使用本系统 |
|------|---------|-----------|
| 查找技术文档 | 翻文件夹、搜文件名 | 直接问"Redis 集群最少几个节点？"→ 秒出答案 + 引用来源 |
| 新人入职 | 花 2-4 周翻阅资料 | 边做边问，系统当导师 |
| 重复性问题 | HR/IT 反复回答 | 员工自助查询，减少 60% 工单 |
| 知识传承 | 老员工离职带走经验 | 文档入库永久留存，AI 随时调用 |

## 二、系统架构

```
┌─────────────────────────────────────┐
│           前端 (Vue 3 + Vite)         │
│  登录 → 仪表盘 → 智能问答 → 文档管理    │
│         → 知识库 → 系统设置            │
├─────────────────────────────────────┤
│        后端 API (FastAPI)             │
│  认证 · 知识库CRUD · 文档上传/解析     │
│  切片/向量化 · 混合检索 · LLM生成      │
├─────────────────────────────────────┤
│          基础设施                      │
│  SQLite / PostgreSQL · 本地 Embedding  │
│  DeepSeek API (LLM) · 文件存储         │
└─────────────────────────────────────┘
```

### 技术栈

| 层次 | 技术 |
|------|------|
| 前端框架 | Vue 3 + Vite + Tailwind CSS |
| 状态管理 | Pinia |
| 后端框架 | Python FastAPI |
| 数据库 | SQLite (开发) / PostgreSQL (生产) |
| LLM 引擎 | DeepSeek Chat API |
| Embedding | BGE-Small-ZH-v1.5 (本地, 512维) |
| 文档解析 | PyMuPDF / python-docx / python-pptx |
| 认证 | JWT (bcrypt + HS256) |

## 三、功能模块

### 3.1 登录认证

- 账号/邮箱 + 密码登录
- JWT Token 鉴权，24 小时有效期
- 登录守卫：未登录自动跳转登录页
- 角色权限：admin / editor / viewer 三级
- 自动聚焦输入框，Enter 快捷登录

### 3.2 仪表盘

| 组件 | 说明 |
|------|------|
| 统计卡片 | 知识库数 / 文档总量 / 今日查询 / 存储用量（数字滚动动画） |
| 查询趋势图 | 7天/30天 SVG 折线图，鼠标悬停显示数值 |
| 知识库排行 | Top 5 知识库排名，彩色进度条 |
| 最近上传 | 最近 5 篇文档列表，含状态标签 |
| 热门问题 | Top 5 高频查询问题 |
| 快捷操作 | 智能问答 / 上传文档 / 新建知识库 / 系统设置 一键直达 |

### 3.3 智能问答

- 自然语言提问 → DeepSeek 生成答案
- 答案附来源引用：文档名 + 分块位置 + 相关度百分比
- 点击来源卡片 → 弹出分块原文查看
- 推荐问题气泡（一键提问）
- 对话历史记录（存 localStorage）
- 知识库范围筛选（可限定只在某个知识库中检索）
- 右侧检索链路面板：向量化 → 向量检索 → 关键词检索 → 融合 → LLM 生成，每步耗时
- Enter 发送 / Shift+Enter 换行

**检索流程：**
```
用户提问 → 关键词提取 → SQLite LIKE匹配 50条
                     → 向量化(本地BGE) → 余弦相似度排序 50条
       → RRF 融合 → 取 Top-5 → 组装Prompt → DeepSeek → 返回答案+来源
```

### 3.4 文档管理

| 功能 | 说明 |
|------|------|
| 上传 | 支持 PDF / Word / PPT / Markdown / TXT，拖拽或点击选择，多文件批量上传 |
| 自动处理 | 上传 → 解析 → 切片 → 入库，全自动同步完成 |
| 预览 | 点击文档 → 弹窗显示解析后的纯文本内容（支持任意格式） |
| 筛选 | 按知识库 / 文件类型 / 状态 三维筛选 |
| 搜索 | 实时搜索文件名 |
| 分页 | 每页 10 条，页码导航 |
| 批量操作 | 多选 → 批量删除 |
| 下载 | 下载原始文件 |

### 3.5 知识库管理

- 卡片网格展示：名称、描述、文档数、切片数、存储大小
- 新建：名称 + 描述 + 部门 + 8 色主题可选
- 编辑：点击编辑图标 → 弹窗修改
- 删除：二次确认弹窗，展示删除用户信息
- 详情抽屉：点击卡片 → 右侧滑出统计 + 文档列表
- 搜索：按名称/描述过滤

### 3.6 系统设置

| 模块 | 内容 |
|------|------|
| LLM 模型 | DeepSeek 配置：API Key + Token + Temperature + 测试连接按钮 |
| Embedding | 模型选择 + 维度 + Batch Size |
| 切片策略 | Chunk Size/Overlap 滑块 + 策略下拉 + 分隔符 |
| 检索参数 | Top-K + 语义权重 α + 阈值 滑块 |
| 用户管理 | 列表/添加/编辑/禁用/启用/删除/重置密码/搜索/筛选/分页 |
| 系统信息 | 版本号 + 框架 + DB + 知识库/文档/用户数 |

### 3.7 全局功能

| 功能 | 说明 |
|------|------|
| 全局搜索 | Ctrl+K / ⌘K 唤起，实时搜索知识库和文档，Enter 跳转 |
| 通知中心 | 文档索引完成/新用户注册/解析失败等消息，已读/未读状态 |
| 用户菜单 | 头像 + 姓名 + 角色 + 系统设置 + 退出登录 |
| 侧边栏 | 可折叠，管理员专属菜单项（系统设置）仅对 admin 角色可见 |

## 四、角色权限

| 角色 | 权限范围 |
|------|---------|
| **admin** (超级管理员) | 全部功能：管理用户、所有知识库、系统设置 |
| **editor** (编辑者) | 创建/编辑/删除自己的知识库，上传/管理文档，使用问答 |
| **viewer** (阅读者) | 仅查看和问答，不可创建/删除知识库和文档 |

权限控制点：
- 前端：侧边栏菜单项按角色过滤，路由守卫拦截非授权访问
- 后端：每个 API 端点校验 `current_user.role`

## 五、部署说明

| 组件 | 最低要求 |

### 5.2 启动后端

```bash
cd backend
pip install -r requirements.txt
python run.py          # 默认端口 8000
```

### 5.3 启动前端

```bash
npm install
npm run dev            # 默认端口 5173
```

### 已知限制

1. 向量检索目前在内存中计算余弦相似度，文档量超过 10 万时需迁移至 Milvus
2. 关键词检索使用 SQLite LIKE，生产环境应切换至 Elasticsearch
3. Embedding 使用 CPU 推理，高并发场景建议 GPU 部署
4. 文档处理为同步执行，大文件上传时会阻塞请求，建议迁移至 Celery 异步
